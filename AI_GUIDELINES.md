# AI Guidelines for Hyperledger Cacti Demos

This document establishes advisory guidelines for the responsible use of artificial intelligence (AI) and large language model (LLM) tooling across all repository activities.

## 1. Individual Responsibility

AI generated code is the **sole responsibility of the committer**. The contributor who submits a pull request is accountable for the correctness, security, licensing compliance, and long-term maintenance of all code in that PR regardless of whether it was written by a human or generated by a tool.

## 2. Disclosure

Code that was generated or assisted by AI **shall be disclosed**. Contributions should include an `Assisted-by` tag in the commit message using the following template:
`Assisted-by: PROVIDER:MODEL_VERSION [TOOL1] [TOOL2]`
**Examples:**

- `Assisted-by: anthropic:claude-3.5-sonnet`
- `Assisted-by: openai:gpt-4o`
- `Assisted-by: google:gemini-2.5-pro`

## 3. Human Review and DCO

All AI-generated output **must be reviewed by a human** before submission.
**AI agents must not add Signed-off-by or DCO tags.** Only humans can legally certify the Developer Certificate of Origin. The human submitter is responsible for reviewing all AI-generated code and adding their own `Signed-off-by` tag.

## 4. Compliance with Project Standards

AI assisted contributions are held to the **same standards** as human-authored contributions. AI tooling does not exempt a contributor from the requirement to submit [Small, Focused Pull Requests](./PULL.md). AI tooling can easily produce large changes, thus contributors must break work into focused, reviewable units.
